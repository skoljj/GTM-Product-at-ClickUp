# The Thesis: Old Game / New Game

Traditional enablement is a content warehouse. Capabilities as a product is a factory with a feedback loop.

---

## Old Game: Traditional Enablement

The traditional model follows a linear path: create content, deliver it through an LMS or email, hope people consume it, and measure completion rates. The metrics that matter — behavioral change, outcome improvement, per-person gap diagnosis — are invisible. There's no return signal. No compounding.

The architecture is a dead end:

```
[Content Library] → [LMS/Email] → [Rep's Inbox] → ???
```

The failure mode: "We shipped 47 playbooks this quarter. Adoption score: unknown. Revenue impact: unclear. But completion rates are strong!"

The metaphor is a **warehouse** — you fill shelves, organize aisles, hope people browse, count foot traffic, and never know if anyone used what they took home.

---

## New Game: Capabilities as a Product

The new model treats capabilities like product features. They're engineered with user research (call analysis, gap data from vaults), shipped through a product stack (orchestrator routes to the right person at the right time via tool connections), and measured by behavioral change — not content consumption.

The architecture is circular:

```
[Knowledge Vault] ←→ [Orchestrator] ←→ [MCP Tools]
       ↕                    ↕                 ↕
  [Gap Analysis]    [AI Agent Layer]    [Behavioral Data]
       ↕                    ↕                 ↕
  [Roadmap]         [Prescriptions]     [Field Evidence]
       ↕                    ↕                 ↕
       └────────── [Feedback Loop] ──────────┘
```

Self-correcting. Compounding.

The success mode: a discovery gap detected via call pattern analysis triggers a coaching intelligence agent to generate a targeted brief. The frontline manager delivers it in a 1:1. The rep's behavior changes within two weeks. Deal velocity improves. The system logs the intervention as effective and deploys similar prescriptions to other reps showing the same gap.

The metaphor is a **factory + operating system** — you engineer capabilities on a product roadmap, ship them through an architecture, measure behavioral change, and iterate based on data. The system gets smarter every cycle.
